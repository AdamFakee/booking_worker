description: Quy trình bắt buộc trước khi thực hiện viết code cho dự án Worker
---

Để đảm bảo hiểu rõ nghiệp vụ (BRD/SRS) và hướng phát triển của dự án, mỗi khi thực hiện thay đổi mã nguồn hoặc thêm tính năng mới, bạn PHẢI thực hiện các bước sau:

1. **Đọc file README.md**: Mở và xem nội dung file `c:\myFolder\intern\cv_blog\worker\README.md` để nắm bắt mục tiêu của Giai đoạn hiện tại (GĐ 1 hoặc GĐ 2), chân dung người dùng và quy trình nghiệp vụ.
2. **Kiểm tra Giai đoạn phát triển**: Xác định tính năng đang làm thuộc Giai đoạn 1 hay Giai đoạn 2 để thiết kế UI/UX phù hợp (ví dụ: GĐ 1 ưu tiên đơn giản, to rõ, kết nối trực tiếp).
3. **Tuân thủ thiết kế UI/UX**:
   - Chữ to, rõ, ít chữ.
   - Ưu tiên hình ảnh minh họa/Icons.
   - Tránh quy trình phức tạp cho Người lao động.
4. **Cập nhật README (nếu cần)**: Nếu có thay đổi lớn về luồng nghiệp vụ trong quá trình code, hãy cập nhật lại vào README để các lần sau vẫn bám sát tài liệu.
